# OA系统集成项目监控设计规格

> 版本：V1.0
> 日期：2026-03-22
> 状态：已批准

---

## 一、需求概述

将AI Flow项目监控系统与公司OA系统对接，获取OA中的工时、审批、财务等数据，实现项目全维度监控分析。

### 1.1 目标

- 从OA系统获取工时、考勤、审批、财务、文档等数据
- 以项目、人员、时间三维关联进行综合分析
- 支持周期性报告、实时看板、预警通知、交互式查询四种输出形式

### 1.2 数据范围

| 数据类型 | 说明 |
|---------|------|
| 工时/考勤 | 员工工作时长、加班、请假 |
| 审批流程 | 请假、报销、合同等审批耗时和通过率 |
| 财务数据 | 项目预算消耗、报销、款项支付 |
| 文档/交付物 | OA中沉淀的项目文档 |

## 二、架构设计

### 2.1 整体架构

```
OA数据库(MySQL/Oracle)
    │
    │ 只读账号 + VPN/堡垒机网络隔离
    ▼
ETL任务层 (Airbyte/DolphinScheduler)
    │
    ▼
数据明细层 DWD
    │
    ▼
数据汇总层 DWS
    │
    ▼
AI Flow应用层
    │
    ▼
输出层 (报告/看板/预警/查询)
```

### 2.2 数据明细层 DWD

| 表名 | 说明 | 主要字段 |
|-----|------|---------|
| `dim_time` | 时间维度表 | date, week, month, quarter, year, is_workday |
| `dim_employee` | 员工维度表 | emp_id, name, dept_id, dept_name, post, entry_date |
| `dim_project` | 项目维度表 | project_id, name, PM, dept, start_date, status, budget |
| `fact_work_hour` | 工时事实表 | id, emp_id, project_id, date, hours, task_type |
| `fact_approval` | 审批事实表 | id, approval_type, emp_id, project_id, submit_time, approve_time, status |
| `fact_expense` | 报销事实表 | id, emp_id, project_id, expense_type, amount, submit_date, status |

### 2.3 数据汇总层 DWS

| 表名 | 说明 | 更新频率 |
|-----|------|---------|
| `dws_project_employee_time` | 项目人员工时汇总 | 每日 |
| `dws_project_cost_weekly` | 项目成本周报 | 每周 |
| `dws_approval_stats_daily` | 审批统计日表 | 每日 |
| `dws_employee_workload_month` | 员工工时负荷月表 | 每月 |

## 三、数据流转

### 3.1 ETL抽取策略

| 数据源 | 抽取方式 | 频率 | 说明 |
|-------|--------|------|------|
| 工时数据 | 全量+CDC增量 | 每日全量 + 工作日增量 | 通过create_time/update_time字段增量 |
| 审批数据 | 增量 | 每日增量 | 按submit_time范围增量 |
| 财务数据 | 增量 | 每日增量 | 按submit_date范围增量 |
| 人员数据 | 全量 | 每周全量 | 人员变动不频繁 |

### 3.2 跨表关联逻辑

```
项目健康度 = f(进度偏差, 成本偏差, 工时投入, 审批效率)

其中：
- 进度偏差 = 实际完成工作量 / 计划完成工作量
- 成本偏差 = 实际报销金额 / 项目预算
- 工时投入 = 实际工时 / 预算工时
- 审批效率 = 平均审批周期 / 标准审批周期
```

## 四、输出形式

### 4.1 周期性报告（周报/月报）

- **触发条件**：每周一生成周报 / 每月1号生成月报
- **数据来源**：DWS 汇总层
- **流程**：AI Flow查询DWS → 项目监控Agent分析 → 生成Markdown报告 → 发送至项目经理

### 4.2 实时看板

- **数据来源**：DWS 实时查询
- **展示内容**：
  - 项目健康度仪表盘（0-100分）
  - 进度/成本/工时三维对比图
  - TOP5风险项目列表
  - 人员负荷热力图

### 4.3 预警通知

| 预警规则 | 风险等级 |
|---------|---------|
| 成本偏差 > 10% | [中] |
| 成本偏差 > 20% | [高] |
| 审批周期 > 3天 | [低] |
| 工时投入超预算 100% | [紧急] |

- **推送渠道**：邮件 / 钉钉 / 企业微信

### 4.4 交互式查询

- **入口**：AI Flow对话界面
- **示例问题**：
  - "3月份哪些项目预算超支了？"
  - "张三最近在哪些项目上投入了工时？"
  - "对比一下Q1三个项目的健康度"
- **实现**：NL2SQL + AI Flow自然语言理解

## 六、关键技术选型

| 模块 | 推荐方案 | 说明 |
|-----|---------|------|
| ETL工具 | Airbyte / DolphinScheduler | 开源，支持多种数据源 |
| 数据仓库 | PostgreSQL / ClickHouse | 轻量级方案 |
| 定时任务 | APScheduler / DolphinScheduler | 报告/预警触发 |
| AI模型 | 现有AI Flow Agent复用 | 项目监控Agent扩展 |
| 对话引擎 | RAG + NL2SQL | 交互式查询 |

---

## 七、安全设计

- **网络隔离**：通过VPN/堡垒机访问OA数据库
- **账号权限**：OA库只读账号，无写入权限
- **数据脱敏**：敏感字段（工资等）在ETL阶段脱敏
- **审计日志**：记录所有数据查询操作
