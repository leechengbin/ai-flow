# 本地综合知识库设计文档

> 本地综合知识库 — 支持企业知识管理、招投标知识库、项目管理知识库
> 版本：V1.0
> 编制日期：2026年3月

---

## 一、功能概述

### 1.1 定位

本地综合知识库是企业级AI系统的核心基础设施，支持多类型文档的统一存储、检索和管理。为标书检查、项目监控、合规审计等业务Agent提供知识支撑。

### 1.2 支持的知识类型

| 类型 | 内容 | 来源 |
|------|------|------|
| 企业制度 | 公司章程、财务制度、人事制度 | 人工上传 |
| 技术文档 | 技术规范、接口文档、架构设计 | 人工上传 / 自动同步 |
| 项目档案 | 项目周报、会议纪要、验收报告 | 自动同步 / 人工上传 |
| 招投标知识 | 招标文件、历史标书、资质文件 | 人工上传 |
| 培训资料 | 培训课件、操作手册、FAQ | 人工上传 |
| 行业知识 | 法规标准、行业报告 | 自动采集 |

## 四、功能详细设计

### 4.1 文档管理

#### 4.1.1 文档上传

| 来源方式 | 描述 | 处理流程 |
|----------|------|----------|
| 人工上传 | 用户通过界面上传PDF/Word | 解析 → 分割 → 向量化 → 存储 |
| 自动同步 | 从OA/项目管理系统自动拉取 | 定时任务 → 增量更新 |

#### 4.1.2 文档分类

```
文档库
├── 公开文档（全员可见）
│   ├── 企业制度
│   ├── 培训资料
│   └── 行业知识
├── 部门文档（部门内可见）
│   ├── 技术部
│   ├── 财务部
│   └── 市场部
└── 项目文档（项目成员可见）
    ├── PROJECT-001
    └── PROJECT-002
```

#### 4.1.3 文档生命周期

```
创建 → 审核 → 发布 → 更新 → 归档
```

| 状态 | 描述 | 可检索 |
|------|------|--------|
| 草稿 | 文档创建，未审核 | 否 |
| 审核中 | 等待审核 | 否 |
| 发布 | 正式发布 | 是 |
| 更新 | 文档更新，新版本待审核 | 旧版本否，新版本待审核 |
| 归档 | 历史版本归档 | 是（仅管理员） |

### 4.2 混合检索

#### 4.2.1 检索流程

```
用户输入查询
    ↓
权限过滤（确定可检索文档范围）
    ↓
并行执行：
├─ 关键词检索（BM25）
└─ 语义检索（向量相似度）
    ↓
结果合并（Reciprocal Rank Fusion）
    ↓
重排序（可选）
    ↓
返回Top K结果
```

#### 4.2.2 检索算法

**BM25（关键词）：**
```python
score_bm25 = BM25F(query, document)
```

**语义检索：**
```python
query_embedding = embed(query)
doc_embedding = embed(document)
score_semantic = cosine_similarity(query_embedding, doc_embedding)
```

**结果融合（RRF）：**
```python
def reciprocal_rank_fusion(results_lists, k=60):
    fused_scores = {}
    for results in results_lists:
        for rank, doc in enumerate(results):
            doc_id = doc['doc_id']
            fused_scores[doc_id] = fused_scores.get(doc_id, 0) + 1 / (k + rank + 1)
    return sorted(fused_scores.items(), key=lambda x: -x[1])
```

#### 4.2.3 检索参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| top_k | 5 | 返回结果数 |
| similarity_threshold | 0.7 | 相似度阈值 |
| keyword_weight | 0.3 | 关键词权重 |
| semantic_weight | 0.7 | 语义权重 |

### 4.3 权限管理

#### 4.3.1 三级权限模型

| 文档类型 | 可见范围 | 可管理角色 |
|----------|----------|------------|
| 公开文档 | 全员（已认证用户） | 知识管理员 |
| 部门文档 | 本部门成员 | 部门管理员、知识管理员 |
| 项目文档 | 项目成员 | 项目经理、知识管理员 |

#### 4.3.2 权限检查流程

```
用户发起检索请求
    ↓
获取用户上下文
├─ 用户部门
├─ 用户角色
└─ 参与项目列表
    ↓
构建权限过滤条件
    ↓
仅返回用户有权限访问的文档
```

## 五、数据流图

```
┌──────────────────────────────────────────────────────────────┐
│                        文档上传数据流                          │
└──────────────────────────────────────────────────────────────┘

用户上传文档
        ↓
文件类型检测 (PDF/Word/TXT)
        ↓
DocumentParser 解析
├─ 提取文本内容
├─ 提取表格
└─ 提取元数据
        ↓
TextSplitter 文本分割
├─ 按段落分割
└─ 保持语义连贯（300-500字/块）
        ↓
Embedding 生成向量
        ↓
权限检查 + 存储
├─ 向量数据库存储
└─ 元数据库存储
        ↓
索引构建完成

┌──────────────────────────────────────────────────────────────┐
│                        文档检索数据流                          │
└──────────────────────────────────────────────────────────────┘

用户输入查询
        ↓
Query Understanding 理解查询
├─ 提取关键词
└─ 语义扩展
        ↓
权限过滤条件构建
        ↓
并行检索
├─ BM25 关键词检索
└─ 向量语义检索
        ↓
结果融合 (RRF)
        ↓
后处理
├─ 去重
└─ 高亮标注
        ↓
返回结果 + 引用来源
```

## 六、错误处理

| 场景 | 处理策略 |
|------|----------|
| 文件解析失败 | 返回错误提示，告知用户支持的格式 |
| 向量化服务不可用 | 降级为纯关键词检索，提示用户"语义检索暂时不可用" |
| 用户无权限访问文档 | 静默过滤，不返回该文档 |
| 检索无结果 | 返回空列表，提示"未找到相关内容，建议尝试其他关键词" |
| 文档内容过长 | 自动截断，标注"内容已截断" |
