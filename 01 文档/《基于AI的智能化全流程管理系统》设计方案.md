## 1 概述

### 1.1 项目背景

随着企业数字化转型加速，传统项目管理方式已无法满足复杂多变的业务需求。本项目旨在构建一个基于AI的智能化全流程管理平台，通过整合项目立项、执行、监控、结项等核心场景，融合财务数据管理与智能分析，为企业提供端到端的智能化解决方案。

### 1.2 项目用途

本系统面向企业内部管理，主要服务于以下业务场景：

#### 1. 项目管理部

- 项目全生命周期跟踪：从立项审批到项目结项全程可视化
- 项目进度监控：实时掌握各项目阶段完成情况
- 项目成本管控：预算编制、执行监控、超支预警
- 项目绩效分析：多维度分析项目投入产出比

#### 2. 财务部

- 项目成本核算：自动归集人工成本、报销费用、采购成本
- 利润分析：实时计算项目利润，支持预算对比
- 财务报表：自动生成项目财务报表、成本分析报告
- 费用合规检查：自动校验费用报销合规性

#### 3. 市场部

- 标书智能辅助：自动检查标书完整性、合规性、技术方案
- 历史案例参考：快速检索相似历史项目案例
- 成本估算辅助：基于历史数据辅助投标报价
- 合同条款检查：自动比对合同关键条款

#### 4. 审计合规部

- 合规自动检查：基于规则引擎自动检测违规行为
- 风险预警：实时监控项目风险指标，自动预警
- 审计报告生成：自动生成阶段性审计报告
- 整改跟踪：跟踪问题整改落实情况

#### 5. 综合管理部

- 知识库建设：统一管理企业知识资产（合同、标书、制度）
- 文档智能检索：语义化检索，快速定位所需文档
- 人员工时管理：记录员工工作投入，支持成本分摊
- 组织绩效分析：多维度分析部门/人员绩效

## 2 项目目标

### 2.1 突破的关键技术

| 序号 | 关键技术 | 技术描述 | 创新点 |
|------|---------|---------|--------|
| 1 | 多Agent协同架构 | 6大Agent（Data/AI Analysis/Finance/Audit/Document/Bidding）分工协作，实现复杂业务场景的智能处理 | Agent间通信协议、任务调度与结果聚合机制 |
| 2 | RAG增强检索 | 结合向量数据库（pgvector）与BM25混合检索，提升知识获取准确率 | 语义分块策略、混合检索权重自适应 |
| 3 | Function Calling | AI模型调用业务系统函数，实现数据查询、计算等操作 | 标准函数定义、参数校验与结果转换 |
| 4 | 智能标书检查 | 基于历史案例和知识库的AI辅助检查，覆盖格式、资格、技术、商务、合规等维度 | 多维度评分模型、风险识别与建议生成 |
| 5 | 自动成本核算 | 基于工时×费率自动计算人力成本，结合报销和采购实现全成本归集 | 成本归集算法、利润实时计算模型 |

```
DRAFT(草稿) → PENDING(待审批) → APPROVED(已审批) → IN_PROGRESS(进行中) → COMPLETED(已完成) → ARCHIVED(已归档)

                    REJECTED(驳回)
```

|------|------|

#### 4.2.3 财务管理

**成本计算公式**

```
项目利润 = 项目收入 - 总成本

总成本 = 人力成本 + 直接费用 + 分摊成本

人力成本 = Σ(工时记录.hours × 人员费率)
直接费用 = Σ(报销金额) + Σ(采购金额)
分摊成本 = 公共费用 × (项目预算 / 总预算)
```

**成本类型**

| 类型代码 | 类型名称 | 说明 |
|---------|---------|------|
| LABOR | 人力成本 | 工时 × 费率 |
| EXPENSE | 报销费用 | 经审批的报销单 |
| PURCHASE | 采购费用 | 采购订单 |
| OVERHEAD | 分摊成本 | 公共费用分摊 |
